# 02 — Data Model, Scoring, SRS (all exact — no interpretation allowed)

## 1. Dictionary file format (`extension/data/dictionary.json`)

Generated by `scripts/build-dictionary.mjs`. A single JSON object mapping **lookup key → entry**:

```json
{
  "食べる": { "r": "たべる", "g": ["to eat"], "l": 5, "f": 640 },
  "退屈":   { "r": "たいくつ", "g": ["tedium", "boredom"], "l": 3, "f": 7200 }
}
```

- Key: the word's primary written form (kanji form if it has one, else kana form). Kana-only words (e.g. きれい) are keyed by kana. Words are ALSO keyed by their kana reading when that kana form is itself common (`re_pri` present) — duplicate value objects are fine.
- `r`: reading in **hiragana**.
- `g`: up to 4 English glosses (strings).
- `l`: level 5..1 (5 = easiest / N5-ish, 1 = hardest). Derived from frequency rank: `f < 1500 → 5`, `< 4000 → 4`, `< 9000 → 3`, `< 18000 → 2`, else `1`. (Optionally overwritten by a real JLPT list — see the script's `--jlpt` flag; this is OPTIONAL and skippable.)
- `f`: frequency rank, integer, lower = more common. From JMdict `nfXX` priority tags: `f = XX * 500 - 250`, then **capped** by curated colloquial tags (`ichi1` → 3000, `spec1`/`gai1` → 5000, `news1` → 10000) because nf ranks are newspaper-derived and undervalue everyday speech. Entries with only second-tier tags (`ichi2` etc.) get `f = 20000`. Entries with no priority tag at all are **excluded** from the dictionary.

Actual size (verified): 30,135 words → 51,362 keys (incl. kana-form aliases), 4.4 MB. **The file is already generated and committed in this repo** — the build script only needs re-running if you want to regenerate it. Loaded once per tab via `fetch(chrome.runtime.getURL("data/dictionary.json"))`.

## 2. Token shape (output of `AVC.tokenizer.tokenize`)

kuromoji's `tokenizer.tokenize(text)` returns objects; we map each to:

```js
{ surface: t.surface_form, base: t.basic_form === "*" ? t.surface_form : t.basic_form,
  reading: kataToHira(t.reading || ""), pos: t.pos, pos1: t.pos_detail_1 }
```

`kataToHira` — VERBATIM:

```js
function kataToHira(s) {
  return s.replace(/[ァ-ヶ]/g, ch => String.fromCharCode(ch.charCodeAt(0) - 0x60));
}
```

## 3. `chrome.storage.local` schema

Four top-level keys. Never store anything else.

```js
// key: "settings"
{ pauseMode: "pause"|"notify"|"off", cooldownSec: 20, maxCardsPerHour: 12,
  targetLevel: 4, autoResumeSec: 0,
  sites: { youtube: true, netflix: true, generic: true } }

// key: "vocab"  — one record per word the user has ever judged or been shown
{ [base: string]: {
    state: "new"|"learning"|"known"|"ignored",
    reading: string, gloss: string,        // denormalized copy (first gloss) for popup display
    level: number, freqRank: number,
    seenCount: number,                     // subtitle appearances observed
    shownCount: number,                    // cards shown for this word
    firstSeenAt: number, lastSeenAt: number,   // epoch ms
    srs: { stage: number, dueAt: number, lapses: number } | null   // only when state === "learning"
} }

// key: "stats"
{ daily: { [yyyy-mm-dd: string]: { met: number, judged: number, reviews: number, watchMin: number } },
  cardTimestamps: number[]   // epoch ms of cards shown in the last hour (pruned on write)
}

// key: "session" is NOT stored — per-tab state lives in memory in main.js
```

### Storage access rules (prevents corruption)

- All writes go through `AVC.storage` helpers; each helper does one atomic read-modify-write of ONE top-level key (`chrome.storage.local.get` → mutate → `set`).
- Date key = local date: `new Date().toLocaleDateString("sv")` (gives `YYYY-MM-DD`). Use this exact idiom everywhere.
- `judgeWord(base, judgment, meta)` semantics:
  - judgment `"know"` → `state: "known"`, `srs: null`.
  - judgment `"learn"` → `state: "learning"`, `srs: { stage: 1, dueAt: now + INTERVALS[1], lapses: 0 }`.
  - judgment `"ignore"` → `state: "ignored"`, `srs: null`.
  - judgment `"review-pass"` → stage +1; if new stage > 5 → `state: "known"`, `srs: null`; else `dueAt = now + INTERVALS[stage]`.
  - judgment `"review-fail"` → `stage = 1`, `lapses += 1`, `dueAt = now + INTERVALS[1]`.
  - Every judgment increments `stats.daily[today].judged` (and `.reviews` for the two review judgments).
  - `meta = {reading, gloss, level, freqRank}` is written on first creation of the record.

## 4. Watch-time tracking

`main.js` starts a 60-second `setInterval` once the adapter is active; each tick where `getVideo()` exists, is not paused, and not ended → `recordWatchTick()` → `stats.daily[today].watchMin += 1`.

## 5. Target-word selection (`AVC.scoring.pickTarget`) — EXACT ALGORITHM

Input: `tokens` (§2), `wordStates` (the `vocab` map), `settings`.

**Step 1 — eligibility filter.** A token survives only if ALL hold:
1. `pos` is one of: `"名詞"` (noun), `"動詞"` (verb), `"形容詞"` (i-adjective), `"副詞"` (adverb).
2. If `pos === "名詞"`, `pos1` is NOT one of: `"代名詞"`, `"数"`, `"接尾"`, `"非自立"`, `"固有名詞"` (pronouns, numbers, suffixes, dependent nouns, proper nouns).
3. `base.length >= 2` OR `base` contains at least one kanji (`/[一-鿿]/.test(base)`).
4. `AVC.dict.lookup(base)` returns an entry.
5. `wordStates[base]?.state` is not `"known"` and not `"ignored"`.
6. The word was not already targeted this session (session-local `Set` in main.js).

Also: for every surviving token AND every token that failed only rule 5/6, increment that word's `seenCount` (via a batched `AVC.storage` call once per line, not per token).

**Step 2 — review priority.** Among survivors where `state === "learning"` and `srs.dueAt <= Date.now()`: if any exist, return the one with the **oldest** `dueAt`, flagged `isReview: true`. Done.

**Step 3 — new-word scoring.** Otherwise score each survivor:

```
freqScore  = 1 - min(entry.f, 20000) / 20000          // 0..1, common → high
levelScore = 1 - Math.abs(entry.l - settings.targetLevel) / 4   // 0..1, at-level → 1
familiarity = min((wordStates[base]?.seenCount || 0), 5) / 5    // seen often → worth teaching
score = 0.45 * freqScore + 0.35 * levelScore + 0.20 * familiarity
```

Return the highest-scoring survivor with `score >= 0.35`, flagged `isReview: false`; else return `null`. Tie-break: lower `f` wins.

## 6. SRS intervals — EXACT

```js
// VERBATIM — index by stage number (stage 0 unused)
const INTERVALS = [0, 4*3600e3, 24*3600e3, 3*24*3600e3, 7*24*3600e3, 21*24*3600e3];
// stage:        -   1: 4 hours  2: 1 day    3: 3 days     4: 7 days     5: 21 days
```

Graduation: passing a review at stage 5 → `known`. Reviews only trigger **in context** (word appears in a subtitle while due) — there is no standalone review UI in v1.

## 7. Pause-engine gates (main.js, evaluated in this order)

1. `settings.pauseMode === "off"` → do nothing.
2. `AVC.overlay.isOpen()` → do nothing.
3. Line identical to the previous processed line → do nothing (subtitle re-renders fire mutations for the same text; also normalize whitespace before comparing).
4. `pickTarget` returned null → do nothing.
5. `isReview` targets **bypass** the cooldown and hourly cap (reviews are precious). New-word targets require: `Date.now() - lastCardAt >= cooldownSec * 1000` AND fewer than `maxCardsPerHour` timestamps within the last hour in `stats.cardTimestamps`.
6. Mode `"pause"` → pause video (remember `wasPlaying`), show card; mode `"notify"` → toast.
7. On card resolution: resume video only if `wasPlaying` was true and the user didn't manually resume already.
