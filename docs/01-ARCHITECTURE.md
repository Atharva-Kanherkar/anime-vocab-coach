# 01 — Architecture

## 1. High-level data flow

```
subtitle DOM change (site adapter, MutationObserver)
        │  raw line text (string)
        ▼
main.js: dedupe consecutive identical lines
        │
        ▼
tokenizer.js (kuromoji, runs IN the content script)
        │  tokens [{surface, base, reading, pos}]
        ▼
dictionary.js: lookup base form → {reading, glosses, level, freqRank}
        │  lookedUpTokens
        ▼
scoring.js: pick ≤1 target word (SRS-due beats new; formula in docs/02 §5)
        │  target | null
        ▼
main.js pause-engine: cooldown / hourly-cap / mode checks
        │
        ▼
overlay.js: pause video, render card (shadow DOM), await user judgment
        │  judgment
        ▼
storage.js: update word state + stats  ──►  chrome.storage.local
                                              ▲
                     popup / options read & write the same storage
```

Everything except the popup/options pages runs in the **content script** context. The background service worker does almost nothing (see §5) — this is deliberate: kuromoji cannot run in an MV3 service worker (`docs/06-PITFALLS.md` §1).

## 3. manifest.json — VERBATIM

```json
{
  "manifest_version": 3,
  "name": "Anime Vocab Coach",
  "version": "0.1.0",
  "description": "Learn Japanese vocabulary from the anime you watch. Pauses on words worth learning and tracks your progress.",
  "permissions": ["storage"],
  "host_permissions": ["*://*.youtube.com/*", "*://*.netflix.com/*"],
  "background": { "service_worker": "background.js" },
  "action": { "default_popup": "popup/popup.html", "default_title": "Anime Vocab Coach" },
  "options_page": "options/options.html",
  "content_scripts": [
    {
      "matches": ["*://*.youtube.com/*", "*://*.netflix.com/*"],
      "js": [
        "vendor/kuromoji.js",
        "src/log.js",
        "src/storage.js",
        "src/dictionary.js",
        "src/tokenizer.js",
        "src/scoring.js",
        "src/overlay.js",
        "src/adapters/youtube.js",
        "src/adapters/netflix.js",
        "src/adapters/generic.js",
        "src/main.js"
      ],
      "run_at": "document_idle",
      "all_frames": false
    }
  ],
  "web_accessible_resources": [
    {
      "resources": ["kuromoji/dict/*", "data/dictionary.json"],
      "matches": ["<all_urls>"]
    }
  ]
}
```

Notes you must respect:
- **Script order in `content_scripts.js` is the dependency order.** All files share one isolated world; later files use globals defined by earlier ones. Do not reorder.
- `all_frames: false` — YouTube and Netflix both render the player in the top frame. (The generic adapter for other sites is wired in M7 via the options page's "enable on this site" flow, still top-frame only in v1.)
- The `generic.js` adapter still ships in the YouTube/Netflix content-script list; it self-deactivates when another adapter matches (see §4 contract).

## 4. The `AVC` global namespace (content-script side)

`src/log.js` is the first of our files to run:

```js
// VERBATIM
window.AVC = window.AVC || {};
AVC.log = (...args) => console.log("[AVC]", ...args);
AVC.warn = (...args) => console.warn("[AVC]", ...args);
```

Every other `src/*.js` file attaches its API to `AVC` and defines nothing else in global scope:

| File | Attaches | Contract |
|---|---|---|
| `storage.js` | `AVC.storage` | `getSettings()`, `getWord(base)`, `judgeWord(base, judgment, meta)`, `recordCardShown()`, `recordWatchTick()`, `getStats()` — all async, exact semantics in `docs/02-DATA-MODEL.md` §3–4 |
| `dictionary.js` | `AVC.dict` | `load()` (fetches `data/dictionary.json` once), `lookup(base) -> entry|null` |
| `tokenizer.js` | `AVC.tokenizer` | `init() -> Promise` (kuromoji builder, memoized), `tokenize(text) -> tokens[]` |
| `scoring.js` | `AVC.scoring` | `pickTarget(tokens, wordStates, settings) -> {token, entry, isReview}|null` |
| `overlay.js` | `AVC.overlay` | `showCard(target, sentence, video) -> Promise<judgment>`, `showToast(target)`, `isOpen()` |
| `adapters/*.js` | pushes into `AVC.adapters` (array) | each adapter: `{name, matches() -> bool, start(onLine) , getVideo() -> HTMLVideoElement|null}` where `onLine(text)` is called for each new subtitle line |
| `main.js` | — | on load: pick the **first** adapter whose `matches()` is true; if none, do nothing (idle, near-zero cost). Then run the pipeline of §1. |

`main.js` must lazy-init: kuromoji and the dictionary are only loaded **after** the adapter reports the first subtitle line, so tabs where the user never enables Japanese subs pay ~0 memory.

## 5. background.js (keep it tiny)

Only two jobs:
1. `chrome.runtime.onInstalled` → seed default settings into storage if absent (defaults table in `docs/00-PRODUCT-SPEC.md`).
2. Listen for message `{type: "avc-badge", count}` from content scripts and set the toolbar badge text to today's judged-word count. No other logic. **No kuromoji, no dictionary, no XHR here — ever** (`docs/06-PITFALLS.md` §1).

## 6. Popup & options pages

Plain HTML/CSS/JS pages. They import nothing from `src/` except `storage.js`-equivalent logic — since extension pages CAN use ES modules, but to keep one storage implementation, `src/storage.js` is written to work in both worlds: it defines `AVC.storage` on `window.AVC` and is included via `<script src="../src/log.js"></script><script src="../src/storage.js"></script>` tags in popup.html/options.html **before** popup.js/options.js. No module syntax anywhere.

## 7. Versions (pinned)

- `kuromoji` npm package: `0.1.2` (latest and final release; its `build/kuromoji.js` is a browser UMD bundle and `dict/` contains 12 `.dat.gz` files).
- JMdict source: `http://ftp.edrdg.org/pub/Nihongo/JMdict_e.gz` (English-gloss variant).
- No other runtime dependencies.
